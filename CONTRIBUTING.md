# Вклад в проект

## Принципы

1. **Модель — это контракт.** Любое изменение `src/gostforge/model/` затрагивает парсер, валидатор и экспортёр одновременно. Меняешь модель — следом обновляй `schema_version` и пиши миграцию для существующих проектов.

2. **Проверки изолированы.** Каждая проверка — отдельный модуль в `src/gostforge/validator/checks/`. Должна работать только с `Model` и `Profile`, ничего не зная о парсере и экспортёре.

3. **Профили — это данные.** Логика не должна быть в YAML-файлах профилей. Если правило слишком сложное для декларации в профиле — пишется как проверка-плагин, а в профиле остаётся только её конфигурация.

4. **Регрессионные тесты обязательны.** Любая новая проверка или эвристика парсера сопровождается фикстурой (`tests/fixtures/`) и тестом. Реальные студенческие работы не коммитятся в публичный репозиторий — только обезличенные минимальные кейсы.

## Стиль кода

```bash
ruff check src tests
ruff format src tests
mypy src
pytest
```

Все четыре команды должны проходить перед коммитом.

## Коммиты

Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`. Тело коммита — на русском или английском, по вкусу.

Примеры:
- `feat(validator): добавить проверку H.06 — заголовок не висит внизу страницы`
- `fix(parser): корректная обработка вложенных таблиц`
- `docs(profiles): пример наследования профиля кафедры от базового`

## Этика проверок

Если правило проверки требует **аннотировать чужой документ** комментариями — комментарии всегда конструктивные, с указанием ГОСТа/пункта профиля. Запрещены: оценочные суждения о работе («плохо», «небрежно»), личные обращения, эмоциональная окраска. Цель — помочь автору исправить, а не отчитать.

## Анонимизация фикстур

Если фикстура основана на реальной работе:
1. Снять метаданные (см. `scripts/strip_metadata.py`, появится в фазе 1).
2. Заменить ФИО, названия организаций, индивидуальные темы.
3. Урезать до минимально необходимого фрагмента, демонстрирующего проблему.
4. Положить в `tests/fixtures/synthetic/` с описанием в комментарии.

Файлы из `tests/fixtures/real/` не должны попадать в git (запрещено в `.gitignore`).
