> Что планируется делать дальше. История уже сделанного —

## Текущее состояние

**119 проверок** в 16 категориях · **35 автофиксеров** · **27 CLI-команд** ·
**6 режимов веб-UI** · **REST API на FastAPI** · **локальная SQLite-БД** ·
**1740+ тестов** (`ruff check`, `ruff format`, `mypy --strict` чисты).

Что уже работает «из коробки» — см. [README.md](../README.md#статус).

## Принципы планирования

1. **Каждая фаза даёт рабочий продукт.** Не «недостроенная новая
   фича», а «работает то, что обещано в прошлой итерации».
2. **Парсер — критический путь.** Без надёжного round-trip всё
   остальное теоретическое.
3. **Профили собираются на реальных кафедрах.** Не выдумываем
   «возможные» профили, добавляем по запросу.
4. **Регрессионные тесты обязательны.** Каждая проверка / эвристика
   парсера / автофиксер сопровождается фикстурой и тестом.
5. **Web-UI — основной интерфейс.** CLI и REST API остаются, но
   приоритет UX-доработок — в Streamlit-режимах. Отдельные
   native-клиенты не разрабатываются — браузер (включая мобильный)
   достаточно.
