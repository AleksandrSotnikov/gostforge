## Кодирование

Каждая проверка имеет код вида `БУКВА.NN`:
- `F` — Formatting (страница)
- `T` — Text (основной текст)
- `S` — Structure (структура работы)
- `H` — Headings (заголовки)
- `I` — Images (рисунки)
- `B` — Tables (таблицы)
- `M` — Math (формулы)
- `L` — Lists (списки)
- `R` — References (литература)
- `C` — Cross-references (перекрёстные ссылки)
- `A` — Abbreviations
- `P` — Appendices (приложения)
- `K` — Headers/footers (колонтитулы)
- `V` — Volume metrics
- `X` — Extra (стиль, лингвистика)
- `U` — Units (единицы измерения, ГОСТ Р 8.000-2015)

**Серьёзность:** `error` (нарушение ГОСТа/профиля, обязательно к исправлению), `warning` (вероятная проблема), `info` (предложение по улучшению).

**Статус:** `✓` — реализовано в текущем релизе; пусто — в roadmap.
Актуальный список реализованных кодов — `gostforge checks`.

## Отключение проверок для отдельных разделов

`LogicalSection.disabled_checks: list[str]` — список кодов, не применяемых
к содержимому раздела. Спецзначение `["*"]` отключает все проверки.
Используется для титульного листа, реферата, приложений — частей работы,
оформляемых по своим правилам.

В builder API:

```python
work("X", year=2026) \
    .section("Титульный лист").paragraph("...").skip_all_checks() \
    .section("Приложение А").paragraph("...").skip_checks("H.01", "T.04", "T.05")
```

В UI (Streamlit-конструктор): в редакторе каждого раздела —
expander «Нормоконтроль раздела» с чекбоксом «Не проверять» и
multi-select по категориям (F/T/H/S/R/B/I/L/C/M/A/K/V/P).

Подробнее — [docs/builder.md](builder.md#отключение-проверок-для-разделов).
