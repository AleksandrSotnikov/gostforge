# Плагины проверок

`gostforge` поддерживает расширение через **пользовательские плагины** —
небольшие Python-файлы, которые добавляют свои проверки в общий реестр
валидатора. Это удобно, когда кафедра или организация хочет
автоматизировать контроль своих локальных требований, не форкая
основной репозиторий.

## Что такое плагин

Плагин — это обычный `.py`-файл с одной или несколькими функциями,
зарегистрированными через декоратор `@register("X.NN")`. Каждая
функция получает на вход модель документа (`Document`) и активный
профиль (`Profile`) и возвращает список нарушений (`list[Violation]`).
То есть плагин ничем не отличается от встроенных проверок — он просто
живёт вне основного дерева пакета.

## Где лежат плагины

Загрузчик автоматически сканирует одну директорию:

- **Linux/macOS:** `~/.gostforge/plugins/`
- **Windows:** `%APPDATA%\gostforge\plugins\`

Поиск нерекурсивный — только `.py` верхнего уровня. Файлы, имена
которых начинаются с `_` (`_helper.py`, `__init__.py` и т. п.),
пропускаются как вспомогательные.

Быстро узнать или создать каталог:

```bash
gostforge plugins dir
```

## Пример: своя проверка `Z.99`

Допустим, кафедра требует, чтобы у каждого документа в `metadata.author`
было указано конкретное название факультета. Создаём
`~/.gostforge/plugins/my_dept_checks.py`:

```python
"""Локальные проверки кафедры XYZ."""

from gostforge.model import Document
from gostforge.profile import Profile
from gostforge.validator.engine import Violation, register


@register("Z.99")
def check_faculty_in_author(document: Document, profile: Profile) -> list[Violation]:
    """Метаданные документа должны содержать упоминание факультета."""
    author = document.metadata.author or ""
    if "ФАКУЛЬТЕТ XYZ" not in author.upper():
        return [
            Violation(
                check_code="Z.99",
                severity="warning",
                message="В авторе документа не указан факультет",
                location="metadata.author",
                suggestion='Добавьте "Факультет XYZ" в поле автора',
            )
        ]
    return []
```

После этого:

```bash
$ gostforge plugins list
Директория плагинов: /home/user/.gostforge/plugins

Найдено файлов: 1
  - my_dept_checks.py

Добавлены проверки: Z.99
```

Проверка `Z.99` появилась в реестре. Теперь её можно включить в
кафедральном профиле через `extends:`:

```yaml
# ~/.gostforge/profiles/dept-xyz.yaml
id: dept-xyz
name: Профиль кафедры XYZ
version: "1.0"
extends: gost-7.32-2017
checks:
  Z.99:
    enabled: true
    severity: warning
```

И запустить:

```bash
gostforge check work.docx --profile dept-xyz
```

## Параметры проверок

Если проверке нужны параметры, они читаются из `profile.checks["Z.99"].params`:

```yaml
checks:
  Z.99:
    enabled: true
    severity: warning
    params:
      required_keyword: "Факультет XYZ"
```

```python
@register("Z.99")
def check_faculty_in_author(document: Document, profile: Profile) -> list[Violation]:
    cfg = profile.checks.get("Z.99")
    keyword = (cfg.params.get("required_keyword") or "") if cfg else ""
    ...
```

Так одна и та же реализация легко переиспользуется в нескольких
профилях с разными значениями параметров.

## Что доступно плагину

Плагин получает доступ ко всем публичным API `gostforge`:

- `gostforge.model.Document` и его компоненты (`Paragraph`, `Table`,
  `Heading`, `PageSection` и т. п.) — см. `docs/architecture.md`.
- `gostforge.profile.Profile` и связанные схемы.
- `gostforge.validator.engine.Violation`, `register`.

Никаких ограничений (sandbox) нет — это обычный Python-код в
интерпретаторе.

## Распространение

Плагин — это один файл. Простейшие способы поделиться:

- **Git-репозиторий кафедры.** Студенты делают `git clone
  https://example.com/dept-checks.git ~/.gostforge/plugins`.
- **Архив на сайте.** Распаковать в директорию плагинов.
- **Прямой файл.** Скопировать `my_check.py` в каталог.

Версионирование плагина — на усмотрение автора. Рекомендуется хранить
версию в docstring модуля и обновлять её при изменениях:

```python
"""Локальные проверки кафедры XYZ.

Версия: 1.2.0
Совместимость: gostforge >= 0.1
"""
```

## Опасности и доверие

**Плагин — это произвольный Python-код, который выполняется в том же
процессе, что и `gostforge`.** Никакой изоляции нет. Это значит, что
вредоносный плагин может:

- прочитать любые файлы пользователя, к которым есть доступ;
- отправить их в сеть;
- модифицировать или удалить файлы.

Поэтому:

1. **Устанавливайте только плагины из проверенных источников** —
   официальных репозиториев кафедры, известных коллег.
2. **Просматривайте код перед запуском.** Особенно сетевые вызовы
   (`urllib`, `requests`), запись в файлы, `os.system`, `subprocess`.
3. **Не запускайте `gostforge` с чужими `.docx` под админ-аккаунтом.**

Если плагин содержит ошибку, она будет залогирована как warning и
плагин просто не загрузится — это не уронит весь нормоконтроль.
Однако ошибки в коде самой проверки (исключения внутри функции,
зарегистрированной через `@register`) уронят `gostforge check` —
плагин должен быть написан аккуратно, с обработкой граничных случаев.

## Отладка

Включите подробное логирование, чтобы увидеть, какие плагины
загружаются:

```bash
PYTHONLOGLEVEL=INFO gostforge plugins list
```

Сообщения вида `Загружен плагин ... из ...` и
`Ошибка загрузки плагина ...` идут через стандартный `logging` модуль
с именем `gostforge.plugins`.
