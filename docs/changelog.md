## 2026-Q2 — фокус на UX конструктора и обратная совместимость

### Streamlit-конструктор: UI основной надписи (штампа)
- Sidebar-секция «Основная надпись (штамп ЕСКД)»: чекбокс включения,
  выбор формы (1/2а), обозначение, организация, литера/лист/листов,
  список ролей. Хранится в `state['title_block']`, применяется при
  сборке (`_apply_title_block_from_state`).
- `document_to_state` отражает обнаруженный при импорте штамп (чекбокс
  включён). 2 теста.

### Конструктор: основная надпись через `WorkBuilder.title_block()`
- Fluent-метод `work(...).title_block(designation=..., organization=...,
  roles=[(роль, фамилия, дата), ...], form=..., stage/mass/scale/sheet/
  sheets_total=...)` задаёт штамп прямо при сборке работы. Без `roles`
  — стандартный набор ролей; наименование по умолчанию = заголовок
  работы. Штамп попадает на `PageSection.title_block` и рендерится при
  экспорте. 4 теста.

### Авто-добавление ГОСТ/ФЗ в список литературы (`gostforge.autocite`)
- Новый модуль `autocite.autofill_references(document)` сканирует текст
  на упоминания стандартов («ГОСТ 7.32-2017», «ГОСТ Р 2.105-2019», …) и
  федеральных законов («… № 152-ФЗ») и добавляет недостающие записи в
  раздел «Список использованных источников» и `Document.bibliography`.
- **Описание по ГОСТ Р 7.0.100-2018**: для известных стандартов и ФЗ
  (встроенный каталог: 7.32, 7.0.100, 7.0.5, 7.1, 2.105, 2.104, 8.000,
  8.417; ФЗ 152/149/273/63/162/44) подставляются официальные названия,
  издатель (Москва : Стандартинформ), год и вид носителя (Текст :
  непосредственный). Для прочих — корректный минимальный скелет без
  выдуманных названий.
- **Идемпотентно**: дедупликация по обозначению/номеру против
  существующих записей — пересборка того же файла не плодит дубликаты
  (устойчиво к различиям тире/дефис/пробелов).
- Доступно как `WorkBuilder.autofill_references()` и чекбоксом в
  Streamlit-конструкторе.
- **Inline-ссылки**: `insert_citations=True` (чекбокс «Проставлять
  inline-ссылки [N]») добавляет `Citation` в первый упоминающий абзац —
  идемпотентно. После сборки UI **уведомляет** списком добавленных
  источников. 14 тестов.

### Стресс-тест крупного документа
- `tests/test_stress_large_document.py` (маркер `slow`): 120 разделов ×
  20 абзацев (~500+ страниц) проходят build → export → parse → validate
  с сохранением числа разделов и щедрыми порогами времени — защита от
  деградации O(n²)/зависаний. Маркер `slow` зарегистрирован в pyproject
  (исключается через `-m 'not slow'`).

### Проверка заполненности перекрёстных ссылок: C.06
- Новая проверка: inline-объекты `CrossRef` и `Citation` должны быть
  заполнены и разрешаться. Пустой `target_id`/`source_id` → `error`
  («заполните ссылку / выберите источник»); ссылка на несуществующий
  элемент/источник → `warning`. Дополняет C.01–C.05 (которые работают
  по текстовым шаблонам) на уровне объектов модели. Всего 119 проверок.

### Мягкая валидация разделителя подписи: I.11 / B.12
- Новые info-проверки: разделитель в подписях рисунков (I.11) и таблиц
  (B.12) сверяется с тире, предписанным `styles.{figure,table}.caption.
  format`. I.03/B.03 принимают «—/–/-» нестрого (error на структуру);
  I.11/B.12 мягко (info) сигнализируют, что фактический разделитель
  (например, дефис) не совпадает с профильным («—»). Точечный формат и
  пустые подписи пропускаются. Всего 118 проверок.
- Хелпер `expected_caption_dash(fmt)` извлекает тире из шаблона.

### Конструктор: рамка листа через `WorkBuilder.border()`
- Fluent-метод `work(...).border(style=..., size_eighth_pt=...,
  offset_from=..., space_pt=...)` включает рамку листа ЕСКД при сборке
  (симметрично `.title_block()`). Без вызова рамка приземляется из
  профиля. Рамка кладётся на `PageGeometry.border` и пишется
  `<w:pgBorders>` при экспорте. 3 теста.

### Парсер: извлечение полей штампа (round-trip)
- `_extract_title_block` теперь восстанавливает не только факт наличия,
  но и поля основной надписи из таблицы footer-а: форма 1 (8 колонок) →
  обозначение/наименование/организация/литера/листы/роли; форма 2а
  (2 колонки) → обозначение и номер листа. Зеркально экспортёру.
- Импорт → правка → ре-экспорт сохраняет содержимое штампа (раньше
  терялось — детектировалось только `enabled`). 2 round-trip теста.

### Нормоконтроль штампа: парсер-детекция + проверка/автофиксер F.08
- Парсер определяет **наличие** основной надписи по таблице в нижнем
  колонтитуле (`_extract_title_block`) → `TitleBlock(enabled=True)`
  (поля штампа обратно не разбираются — только детекция).
- **Проверка F.08**: если профиль требует штамп
  (`styles.page.title_block.enabled`), его отсутствие → `error`; если не
  требует — молчит (у многих специальностей штампа нет).
- **Автофиксер F.08**: проставляет штамп из профиля.
- F.08 включён в base-профиль (наследуется ЕСКД); бездействует, пока
  штамп в профиле не включён. Всего 116 проверок, 35 автофиксеров.

### Основная надпись (штамп ЕСКД, ГОСТ 2.104)
- Новые типы модели `TitleBlock` и `TitleBlockRole`, поле
  `PageSection.title_block` (`None` по умолчанию). `SCHEMA_VERSION` → 0.6.0.
- Экспортёр рендерит штамп таблицей в нижнем колонтитуле (повторяется
  на каждой странице): **форма 1** (заглавный лист, 8 колонок 185 мм —
  обозначение, наименование, роли Разраб./Пров./…, литера/масса/масштаб,
  лист/листов, организация) и **форма 2а** (узкая, для последующих
  листов). Модуль `exporter/title_block.py`.
- Штамп задаётся профилем (`styles.page.title_block`, схема
  `TitleBlockProfile` с набором ролей по умолчанию): у каждой
  специальности — свой штамп/форма/организация или его отсутствие
  (`enabled: false`). Приземляется на модель в
  `_sync_page_section_with_profile`. Наименование по умолчанию = заголовок
  работы.
- В профиль ЕСКД добавлен блок `title_block` (выключен по умолчанию).
- **Редактор профиля (UI)**: раздел «Страница» получил блок «Основная
  надпись (штамп)» — чекбокс включения, выбор формы (1/2а),
  организация, список ролей. Так каждая специальность настраивает штамп
  (или его отсутствие) без правки YAML.
- 7 тестов; документация — `docs/page-sections.md`.
- Геометрия граф приближённая (не пиксельная форма), обратный парсинг
  штампа пока не делается.

### Рамка листа ЕСКД (ГОСТ 2.104) — `<w:pgBorders>`
- Новый тип модели `PageBorder` и поле `PageGeometry.border`
  (`None` по умолчанию — back-compat). `SCHEMA_VERSION` → 0.5.0.
- Экспортёр пишет `<w:pgBorders>` в `sectPr` (`_apply_pg_borders`),
  парсер читает обратно (`_extract_pg_borders`) — round-trip без потерь.
- Рамка задаётся профилем (`styles.page.border`, схема
  `PageBorderProfile`): у каждой специальности/кафедры — своя рамка
  или её отсутствие (`enabled: false`). Профиль «приземляется» на
  модель при экспорте в `_sync_page_section_with_profile`.
- В профиль ЕСКД (`gost-r-2.105-2019.yaml`) добавлен блок `border`
  (выключен по умолчанию, с комментарием как включить).
- Документация — `docs/page-sections.md` (раздел «Рамка листа»).
- **Проверка F.07 + автофиксер F.07** — нормоконтроль рамки: если
  профиль требует рамку (`border.enabled`), её отсутствие → `error`,
  расхождение стиля/толщины → `warning`; фиксер проставляет рамку из
  профиля. Если профиль рамку не требует — проверка молчит (отсутствие
  рамки у такой специальности — норма). Всего 115 проверок, 34
  автофиксера.
- **Редактор профиля (UI)**: раздел «Страница» получил блок «Рамка
  листа (ЕСКД)» — чекбокс включения + стиль/толщина/цвет/отступ. Так
  каждая специальность настраивает свою рамку (или её отсутствие) без
  правки YAML.
- Основная надпись (штамп) — следующий этап.

### Автофиксеры нумерации и колонтитулов секций K.02/K.03/K.04/K.06
- **K.02** — отключает номер страницы на титульном листе
  (`page_numbering.visible = False` для секций `type == "title"`).
- **K.03** — задаёт стартовую страницу основной части
  (`start_mode = "start_at"`, `start_value` из профиля, по умолчанию 3).
- **K.04** — убирает сбросы сквозной нумерации (`restart` → `continue`),
  сохраняя рестарт в приложениях, если он разрешён профилем.
- **K.06** — отвязывает колонтитул секции от предыдущей
  (`link_to_previous` → False), уважая `exclude_types` профиля.
- Все — безопасные правки метаданных вёрстки (не текста),
  симметричны проверкам в `validator/checks/page_sections.py`. Новый
  модуль `fixer/fixers/page_sections.py`.
- **Итог:** 33 автофиксера (было 29).

### Автофиксеры единиц измерения U.02/U.03 и формата нумерации F.05
- **U.02** — знак препинания между числом и единицей измерения
  заменяется на неразрывный пробел (`10.кг` → `10 кг`, `50,%` → `50 %`).
- **U.03** — точка после единицы измерения СИ убирается
  (`10 кг.` → `10 кг`). Зеркалит контекстную эвристику проверки U.03:
  «1990 г.» (год) и «12 с.» (страница) не трогаются.
- Оба фиксера переиспользуют регулярки парных проверок из
  `validator/checks/units.py`, работают в пределах одного TextRun.
- **F.05** — формат номера страницы приводится к профилю (по умолчанию
  arabic). Применяется только к секциям с видимой нумерацией; меняется
  стиль глифа, не сам текст. Зеркально проверке F.05.
- **Итог:** 29 автофиксеров (было 26).

### Парсер + экспортёр многоуровневой шапки таблицы
- Парсер распознаёт `<w:tblHeader/>` и раскладывает строки шапки по
  `Table.headers` (последняя) и `Table.extra_header_rows`

- Маркер «Продолжение таблицы N» автоматически фильтруется при
  парсинге — экспортёр добавит его обратно при необходимости.
- Координаты `merges` корректно сдвигаются.
- SCHEMA_VERSION → 0.4.0 (`Table.extra_header_rows`).

### Live-нормоконтроль: клик «→ К разделу»
- Каждое нарушение в live-панели конструктора получило кнопку
  «→ К разделу», которая активирует соответствующий top-level раздел.
- Маппинг `validator_section_id → state_index` строится через

  корректно проверяет ячейки многоуровневой шапки.

### Конструктор: фичи нумерации, ширины и подписей
- **Высота рисунка**: `FigureStyleProfile.max_height_cm` (default 22 см).
  Экспортёр жмёт картинки и по высоте, не только по ширине, — подпись
  не съезжает на следующую страницу.
- **Схемы нумерации рисунков/таблиц**: `figure.numbering` /
  `table.numbering` (continuous | by_chapter). В приложениях всегда
  буквенно (А.1, Б.1) независимо от режима. WorkBuilder детектит
  «Приложение X» в heading и поддерживает per-chapter счётчики.
- **Повтор шапки таблицы** при переносе (`TableStyleProfile.repeat_header`,
  default True) — экспортёр ставит `<w:tblHeader/>` на header-row.
  Опционально `continuation_caption=True` — добавляет первой строкой
  «Продолжение таблицы N» (merged по колонкам, тоже tblHeader).
- **Многоуровневая шапка таблицы** (`Table.extra_header_rows`):
  «Группа 1 | Группа 2» сверху + «Подзаг A | B | C | D» снизу +
  данные. В web-редакторе поле «Доп. шапка» с авто-склеиванием
  пустых ячеек (`Группа 1||Группа 2|` → две группы по 2 колонки).
- **Профильный формат подписи** (`caption.format`) теперь респектится
  билдером — раньше формат хардкодился, и собственная же проверка
  I.05/B.05 ругалась на несоответствие.

### Багфиксы конструктора
- **Дублирование «Таблица N — Таблица N — ...»** при повторной вставке
  .docx: `_strip_caption_prefix` теперь распознаёт все формы
  разделителя (em-dash, en-dash, дефис, точка, двоеточие, NBSP,
  многоуровневую нумерацию) и применяется итеративно.
- **«Прилипание» состояния виджетов** при удалении/перемещении блока:
  стабильный per-block `id` (uuid4-hex) вместо позиционного индекса
  как ключ Streamlit-виджетов. Расширено на все уровни: блоки, run-ы,
  подразделы (level 2), пункты (level 3), top-level секции.
  Свежие id у новых сущностей теперь uuid4 (а не позиционные
  «sec-N», «sub-X-Y», которые могли пересечься после удаления).

### Автофиксеры формата и геометрии страницы
- **F.01/F.02/F.03** — поля → профиль (допуск 0.5 мм), формат бумаги

- **H.01/H.02** — формат заголовков 1–2 уровня (шрифт/кегль/жирность/

  не меняются.
- **T.01/T.02/U.01** — шрифт и кегль основного текста, неразрывный
  пробел перед единицей СИ.
- Все правят только ЯВНЫЕ run-атрибуты, наследуемые значения
  не трогают, переиспользуют предикаты соответствующих проверок.
- **Итог:** 26 автофиксеров (было 18).

## 2026-Q2 — расширение конструктора и Streamlit-UI

- Конструктор: титульный лист (ручная вставка), приложения с

- Категория `U.*` (единицы измерения СИ, ГОСТ Р 8.000-2015).
- R.14 (формат DOI/URL).
- X.05 (терминологическая консистентность), X.06–X.08 (стилистика:
  канцеляризмы, длинные предложения, повторы слов).

### Парсер: глубокая поддержка OOXML
- Merge-cells таблиц (`<w:vMerge>`/`<w:gridSpan>`).
- Hyperlinks (`<w:hyperlink>`).
- Сноски (`footnotes.xml`).
- Textboxes (`<w:txbxContent>`).
- Style-cascade (font/size/bold/italic/color наследуются от Heading{N}
  → Normal даже без явных run-атрибутов).

### CLI: версионирование и diff
- `state-versions` (список и восстановление), `diff-state` (summary
  и unified), `convert` (DOC→DOCX через LibreOffice).
- GitHub Action для PR (`pr-docx-check.yml`).
