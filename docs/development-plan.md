## Приоритет: браузерный интерфейс (удобство и многофункциональность)

Главный фокус — сделать веб-UI (`gostforge ui`) основной точкой входа:
самодостаточной, удобной и закрывающей весь цикл «собрал → проверил →
выгрузил». Режимы: Нормоконтроль · Конструктор · Редактор профиля ·
История · Документация.

| Задача | Описание | Сложность | Статус |
|---|---|---|---|
| **Редактор профиля форматирования** | Отдельный режим: все параметры стиля (страница/текст/заголовки/таблицы/рисунки/списки), проверки, метаданные; сохранение как снимок или наследник (`extends`); список и удаление кастомных профилей. | M | ✅ |
| **Экспорт из конструктора во все форматы** | DOCX / PDF-превью / Markdown / HTML — кнопки скачивания в одном месте. | S | ✅ |
| **Структурные шаблоны разделов** | Титульный лист (место под ручную вставку), приложения с авто-буквами (А, Б, В… по ГОСТ), содержание (TOC). | S | ✅ |
| **Картинки внутри state** | Изображения вшиваются в state (data-URI) — переживают сохранение/новую сессию; вставка из UI и при импорте .docx. | S | ✅ |
| **Live-нормоконтроль в конструкторе** | Сводка нарушений по текущему state без выхода из редактора. | S | ✅ |
| **Стартовый дашборд / онбординг** | Режим «Главная»: обзор режимов, быстрый старт, шпаргалка по ГОСТ, метрики. | S | ✅ |
| **Кнопка «Собрать каркас по ГОСТ»** | Один клик — весь структурный набор: титульник → содержание → введение → главы → заключение → список → приложение. | S | ✅ |
| **Чек-лист готовности работы** | Панель «Готовность работы»: чек-лист структурных элементов + навигация «Перейти к разделу». | S | ✅ |
| **Документация по UI** | Workflow всех режимов в `docs/builder.md` + секция в README. | S | ✅ |
| **Адаптивная ширина** | `layout=wide`, page_icon, развёрнутый sidebar. | S | ✅ |
| **Клик по нарушению → переход к разделу** | Из live-нормоконтроля прыгать к проблемному разделу/блоку (частично: навигация есть в «Готовности работы»). | M | 🟡 |
| **Цветовая тема (тёмная/светлая)** | Требует механизма доставки config.toml для установленного пакета. | S | ⬜ |
| **Drag-and-drop порядка разделов/блоков** | Через `streamlit-sortables` (новая зависимость). | M | ⬜ |
| **Inline-WYSIWYG абзаца** | `streamlit-quill` или аналог (новая зависимость). | M | ⬜ |
